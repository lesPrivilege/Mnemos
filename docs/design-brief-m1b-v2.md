# Design Brief v2 — Mnemos 視覺重定調

本文件取代 design-brief-m1b.md 及其補遺（兩份留檔不再生效）。隨附輸入：
`docs/design-tokens.md`（讀第 1-6 節了解現狀；其第 8 節硬約束**已被本文件
修訂**，以本文件為準）、`design/handoff/current-state-atlas.html`（現狀圖譜,
~24 幀）、應用 icon 截圖。design/ 下其餘舊 prototype 不在交付範圍,不要參考。

## 任務

Mnemos：AI-native 學習工具（SM-2 閃卡 + 題庫 + 閱讀），學習內容由 LLM 在應用
外產出，應用是其最佳消費端。Local-first、無後端、長期日常使用。現有界面是兩個
月前的產物，其視覺語彙（暖米底、赭石、mono 雙語標籤）已隨同類 AI 生成界面的
泛濫而折舊為模板味。你的任務：把它重畫成一件**克制但有承載力的經典單品**——
十年不過時的那種。這是一次真正的重定調，不是換膚。

## 不變項（全部約束就這六條）

1. **真內容**：一切幀用 atlas 中的真實功能與真實文案，不虛構功能。閱讀模塊的
   示例文本用自撰中性學術段落，不用真實書摘。
2. **icon 構圖是身份錨**：形不變,色將從你的新 token 重導出。方向若讓 icon
   顯得格格不入,錯的是方向。
3. **中文排版是主戰場**：正文節奏、字重層級、標點、留白。參照系 iA Writer /
   Bear / Things——無可模仿的風格特徵,只有難以模仿的執行精度。
4. **反俗套**：禁用 `中文 · ENGLISH` mono 標籤語彙（被系統性刪除的清單見
   tokens 文檔第 6 節,重新引入需逐條書面理由）；不做裝飾性漸變堆砌；不仿真
   iOS 原生控件外觀；打磨度允許不均勻——核心路徑（複習/閱讀/答題）98 分，
   次要頁面敢於樸素。
5. **導航架構已定案**：底部 tab bar（iOS 材質向液態玻璃靠、Android 傳統底欄,
   形態同一）；三層深度規則——L0 模式根 tab 常駐,L1 瀏覽層 push 進入 tab 隱藏,
   L2 沉浸層（複習/答題/閱讀會話）全屏接管 chrome 全隱。轉場語彙按層綁定。
6. **token 名與三層結構不變**（primitives / scales / surfaces，名錄見 tokens
   文檔第 1 節）。**所有值全面開放重提案**——色、字階、間距、圓角、陰影、動效
   檔位皆可推翻重來,雙主題成對給出，語義色保持可讀對比。缺 token 就提案新增,
   不准內聯字面值繞過。

## 兩段式流程

**Phase A — 發散。** 產出若干塊方向板（幾塊、每塊幾幀由你判斷，以能支撐論證
為準；建議覆蓋 Home、複習卡片、閱讀正文中至少兩個場景）。每塊帶完整的
色彩/字階/密度提案和一段設計論述：這個方向是什麼、拒絕了什麼、憑什麼十年
不過時。方向之間必須真正互異。動效可演示，標注將來收斂的 motion 檔位。

**Gate — 人工選向。** 品味決策在人。

**Phase B — 收斂。** 被選方向先形式化為完整 token 值提案（沿用現有名，逐值
給出，雙主題），確認後成為新的硬約束；然後出全量 prototype：atlas 全部頁面
與狀態 + 三個前瞻頁面（導圖視圖：大綱⇄導圖切換、節點三檔掌握度着色；首開
onboarding：三模塊引導 + 示例內容包入口；內容包瀏覽頁：列表 + 詳情）+
「同一頁面 × 手機底欄 / iPad sidebar」兩殼分離證明各一幀。tokens 文檔第 7 節
的四項雙語遺留（HeroSection label / Home tab .en / Activity stat 標籤 /
Reader TOC）在 Phase B 逐項裁決並註明理由。

**交付格式**：單文件 HTML、內聯 token 定義、雙主題切換、手機幀 390px、每幀帶
標題與裁決註釋。Phase A 允許演示性 JS；Phase B 除主題切換外全靜態。
